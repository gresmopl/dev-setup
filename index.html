Konfiguracja środowiska developerskiego

Windows 11, krok po kroku • gresmopl

0 / 10 kroków
01 Git

System kontroli wersji. Wymagany do klonowania repozytoriów i pracy z GitHub.

Pobierz Git for Windows
Podczas instalacji zaznacz "Git from the command line and also from 3rd-party software". PATH ustawia się automatycznie.

Ustaw globalną konfigurację po instalacji:

PowerShell
git config --global user.email "gresmo.tlen.pl@gmail.com"
git config --global user.name "gresmopl"
git config --global push.autoSetupRemote true
git config --global init.defaultBranch main
02 Node.js LTS

Środowisko uruchomieniowe JavaScript. Wymagane do pnpm, Claude Code, Codex CLI i projektów webowych.

Pobierz Node.js LTS
Pobierz wersję LTS (nie Current). PATH ustawia się automatycznie podczas instalacji.
PowerShell — weryfikacja
node --version   # powinno pokazać v20.x.x lub nowszy
npm  --version
03 pnpm

Szybszy menedżer pakietów npm. Używany we wszystkich projektach webowych.

PowerShell
npm install -g pnpm
04 GitHub CLI

Narzędzie GitHub w terminalu. Wymagane do klonowania prywatnych repozytoriów.

PowerShell (jako Administrator)
winget install GitHub.cli
PowerShell — logowanie
gh auth login   # wybierz GitHub.com > HTTPS > Login with web browser
05 Klucz SSH dla GitHub

Umożliwia push/pull bez wpisywania hasła. Wymagany do klonowania prywatnych repozytoriów przez Git.

PowerShell — generowanie klucza
ssh-keygen -t ed25519 -C "gresmopl" -f "$env:USERPROFILE\.ssh\id_ed25519_github"
PowerShell — konfiguracja SSH
Add-Content "$env:USERPROFILE\.ssh\config" "`nHost github.com`n  HostName github.com`n  User git`n  IdentityFile ~/.ssh/id_ed25519_github"
PowerShell — wyświetl klucz publiczny
Get-Content "$env:USERPROFILE\.ssh\id_ed25519_github.pub"
Wejdź na github.com → Settings → SSH and GPG keys → New SSH key i wklej wyswietlony klucz.
PowerShell — weryfikacja
ssh -T git@github.com   # powinno pokazać: Hi gresmopl!

09 Zmienne PATH

Sprawdź że wszystkie narzędzia są dostępne w terminalu.

PowerShell — weryfikacja wszystkiego
git    --version
node   --version
pnpm   --version
gh     --version
claude --version

Jeśli coś nie działa — sprawdź PATH ręcznie:

NarzędzieTypowa ścieżka
Node.js / npmC:\Program Files\nodejs\
pnpmC:\Users\{user}\AppData\Local\pnpm\
GitC:\Program Files\Git\cmd\
ghC:\Program Files\GitHub CLI\
Jak dodać do PATH: Win+R → sysdm.cpl → Zaawansowane → Zmienne srodowiskowe → Path → Edytuj
10 Restore środowiska kluczowy

Sklonuj repozytoria i skonfiguruj Claude Code automatycznie.

Wymaga zalogowania do GitHub SSH (krok 05) i gh CLI (krok 04).
PowerShell (jako Administrator)
git clone git@github.com:gresmopl/knowledge-infra.git C:\tmp\infra
C:\tmp\infra\scripts\restore-windows.ps1

Skrypt automatycznie klonuje wszystkie repozytoria, kopiuje konfigurację Claude Code i instaluje pluginy.
